# v0.2.0 - reliability-first contracts, MCP, and opt-in LLM

## What's inside
- Large internal refactor: `cli.py`, `actions.py`, and `understand.py` are now split into smaller modules with cleaner orchestration seams.
- Versioned machine-readable contracts now cover generated artifacts, CLI JSON output, and MCP read/write tool responses.
- Experimental MCP support now includes `status`, `check`, `detect`, `understand`, `init`, `update`, and `pack`.
- Experimental `--llm-enhance` remains opt-in only and falls back to local-only generation on missing credentials, timeout, or provider failure.
- Added stronger regression coverage: schema snapshots, JSON contract tests, LLM fallback tests, MCP protocol tests, property tests for markers, and smoke/runtime tests under `tests/`.

## Safety model (why it won't wreck your repo)
- Marker-only updates: if `AGENTS.md` / `RUNBOOK.md` have `<!-- AGENTSGEN:START ... -->`, agentsgen updates only inside marker blocks.
- No markers = no touch: if markers are missing, agentsgen writes `AGENTS.generated.md` / `RUNBOOK.generated.md`.
- Preview-first: supports `--dry-run` and `--print-diff`.
- MCP write tools now expose explicit `write_policy` metadata so callers can distinguish `apply`, `dry-run`, and `check` modes.

## GitHub PR Guard (if applicable)
- `agentsgen-guard` still fails PRs when generated sections drift or files are missing.
- Pack drift remains enforceable in the same guard job via `pack_check: "true"`.

## Quickstart

```bash
pipx install git+https://github.com/markoblogo/AGENTS.md_generator@v0.2.0

agentsgen init . --defaults --autodetect
agentsgen pack . --autodetect
agentsgen check . --all --format json
```

## Known limitations
- LLM enhancement currently enriches narrative sections only; built-in providers do not yet vary output by `model`.
- MCP coverage is contract- and protocol-level, but the stdio server is still intentionally minimal and experimental.
- Public-site analysis surfaces (`analyze`, `meta`) are CLI-first and not yet exposed as first-class lab pages.
