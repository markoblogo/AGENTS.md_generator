## Safety model

`agentsgen` still follows the same three write modes:

1. Missing file: create it.
2. Managed file with markers: patch only inside markers.
3. Existing file without markers: leave it alone and write a `*.generated.md` sibling.

This release changes positioning and release hygiene, not the core write contract.

## Quickstart

```sh
pipx install git+https://github.com/markoblogo/AGENTS.md_generator.git
agentsgen init . --defaults --autodetect
agentsgen status .
agentsgen check . --all --ci
```

PyPI trusted publishing is wired in CI. Once the package is published on PyPI, the install path becomes:

```sh
pip install agentsgen
```

## Known limitations

- PyPI visibility still depends on the first successful trusted-publishing release completing against PyPI.
- GitHub release automation also depends on repo-level Actions permissions that allow workflow-created PRs.
- `agentsgen` stays intentionally conservative in ambiguous repos and may leave some command blocks empty until you configure them explicitly.
