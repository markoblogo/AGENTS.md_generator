# v0.4.0 - readiness report, safe fix loop, and fleet scan

## What’s inside

- Added `agentsgen check --report` with an Agent Readiness Score, readiness level, per-surface status, and recommended remediation commands.
- Added conservative `agentsgen fix` remediation for marker-managed docs, pack artifacts, and README snippets.
- Added stable `agentsgen fleet scan` for read-only team rollout reports across many repositories.
- Added a versioned `fleet_scan_report` JSON contract and schema snapshot.
- Updated the product docs, landing copy, `llms.txt`, and public integration discovery with the new readiness/fleet loop.

## Safety model

The write contract is unchanged:

1. Missing file: create it.
2. Managed file with markers: patch only inside marker blocks.
3. Existing file without markers: leave it alone and write a `*.generated.md` sibling.

`agentsgen fleet scan` is read-only. It walks git repos, builds dry-run plans, and reports next actions without writing to scanned repositories.

## Quickstart

```sh
pipx install agentsgen
agentsgen init . --defaults --autodetect
agentsgen check . --all --report
agentsgen fix . --all --dry-run --print-diff
```

Team rollout:

```sh
agentsgen fleet scan ~/code --max-depth 2
agentsgen fleet scan ~/code --format json
```

## Known limitations

- `agentsgen fix` is intentionally conservative: it does not invent commands, rewrite unmarked docs, or mutate repo code.
- `agentsgen fleet scan` reports and plans only; bulk multi-repo writes are intentionally out of scope for this release.
- Ambiguous monorepos may still need explicit `.agentsgen.json` commands before the readiness score becomes meaningful.
