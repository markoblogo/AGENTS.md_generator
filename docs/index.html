AGENTS.md Generator

Generate and maintain AGENTS.md / RUNBOOK.md safely with markers, diffs, conservative autodetect, and zero repo damage.
View on GitHub Get started Why it’s safe

Problem

AI agents are fast. Repo context is not. Most repos either have no agent docs, or they’re outdated, inconsistent, and easy to break.
  • How to run tests / lint / build
  • What “done” means
  • What not to touch

Solution

agentsgen creates and updates agent docs using a safety-first model: updates only inside markers, otherwise generates a sibling file.
  • Markers + patch-by-range updates
  • Diff-first workflow (--dry-run, --print-diff)
  • Auto-detect (no AI)

Safety model

  • Updates are limited to marked sections.
  • --dry-run and --print-diff are first-class.
  • If the file has no markers, the original stays untouched: AGENTS.generated.md.
Three file modes: no file (create) · markers exist (patch) · no markers (write sibling).

Quickstart

pipx install git+https://github.com/markoblogo/AGENTS.md_generator
cd your-repo
agentsgen init --autodetect
agentsgen update --dry-run --print-diff
agentsgen update

Auto-detect (no AI)

Priority rules:
  • Makefile (dominant)
  • package.json scripts
  • Python heuristics (uv / poetry / pytest / ruff, etc.)
  • Otherwise: minimal baseline (no hallucinated commands)

Known limitations

  • Monorepos may be detected as mixed, and commands can be left empty intentionally.
  • Unusual workflows should be defined explicitly in .agentsgen.json.
  • This generates process guidance, not a full developer handbook.
Made for people who want agents to be useful without turning documentation into a second job.
