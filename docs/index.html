Quickstart 
pipx install git+https://github.com/markoblogo/AGENTS.md_generator
cd your-repo
agentsgen init --autodetect
agentsgen update --dry-run --print-diff
agentsgen update
# optional
agentsgen pack --autodetect --check
In CI, run agentsgen check (and optionally pack --check) to fail drift early.

PR Guard + Pack drift check Recommended

Fail PRs when agent docs drift. Optionally enforce LLMO pack drift in the same job.

Why it matters

  • Fail PRs when generated docs drift.
  • pack_check: "true" adds agentsgen pack --check.
  • Marker-only ownership stays intact for handcrafted docs.
Works on forks (best-effort comments).
Workflow 
name: agentsgen guard + pack check

on:
  pull_request:
  push:
    branches: [ main ]

permissions:
  contents: read
  pull-requests: write

jobs:
  agentsgen:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: markoblogo/AGENTS.md_generator/.github/actions/agentsgen-guard@v0.1.2
        with:
          comment: "true"
          files: |
            AGENTS.md
            RUNBOOK.md
          pack_check: "true"
          pack_format: "json"
If you do not want PR comments, set comment: "false".

Safety model

Safe to re-run: generated sections are owned; everything else stays yours.

1

Create docs with section markers.

3

Markers make updates idempotent and reviewable.

Auto-detect (no AI)

Priority rules. Conservative defaults.

  • Makefile (dominant)
  • package.json scripts
  • Python heuristics (uv / poetry / pytest / ruff, etc.)
  • Otherwise: minimal baseline (no hallucinated commands)
No command execution. No network. No AI inference.

LLMO Pack (optional)

AI-readable docs bundle generated with the same safety model.

  • llms.txt (or LLMS.md)
  • docs/ai/* (how-to-run, how-to-test, architecture, data-contracts)
  • SECURITY_AI.md
  • CONTRIBUTING_AI.md
  • README_SNIPPETS.md
Commands: agentsgen pack --autodetect and agentsgen pack --autodetect --check.
If markers are missing, pack writes sibling *.generated.* files instead of overwriting originals.

FAQ

Short answers.

What is AGENTS.md?
A repo-level contract for coding agents: setup, commands, guardrails, and “done” criteria. Think “README for agents”.
Will it overwrite my existing docs?
By default, no. Without markers it writes a sibling *.generated.md file and leaves your originals untouched.
What is the LLMO Pack?
A small bundle of AI-readable docs (run/test/architecture/contracts plus policies) that can be enforced via pack --check.
Does it work for monorepos?
It can detect mixed; if unsure, it stays conservative and may leave commands empty intentionally.

Known limitations

Conservative by design.

  • Monorepos may be detected as mixed; commands can be empty by design.
  • Unusual workflows should be set explicitly in .agentsgen.json.
  • This generates process guidance, not a full developer handbook.
