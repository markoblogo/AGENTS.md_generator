How it fits

Three layers. Clear boundaries.

agentsgen

  • Owns repo-scoped agent context: AGENTS.md, RUNBOOK.md, pack docs, command manifests, repomap artifacts.

ID

  • Owns portable human context: preferences, freshness, privacy, handshakes, and cross-tool continuity.

SET

  • Owns workflow execution: CI entrypoint, orchestration, registry-driven flows, and hook timing.

Bridge

  • docs/ai/id-context.json is the repo-local handoff surface that lets ID meet repo context without guesswork.

Quickstart 
pipx install git+https://github.com/markoblogo/AGENTS.md_generator
cd your-repo
agentsgen init --preset nextjs
agentsgen pack . --autodetect
cat agents.entrypoints.json
cat docs/ai/id-context.json
agentsgen check . --all --ci

The philosophy

Agentsgen is built for one goal: less babysitting, fewer hallucinations, lower token burn.

  • No text for the sake of text. If it doesn’t prevent mistakes, it doesn’t belong in always-loaded context.
  • Instrumented truth beats prose. Prefer check/status/pack --check over “hope this CLAUDE.md stays true”.
  • Progressive disclosure scales. Keep global rules small; pull deeper docs only when the task needs them.
Rooted in: ETH “Do Context Files Help?” + Codified Context (layered memory).

Safety model

Safe to re-run: generated sections are owned; everything else stays yours.

1
File missing
Create docs with section markers.
2
Markers exist
Update only inside marker ranges.
3
No markers
Never overwrite; write sibling *.generated.md.
Markers make updates idempotent and reviewable.

Auto-detect (no AI)

Priority rules. Conservative defaults.

  • Makefile (dominant)
  • package.json scripts
  • Python heuristics (uv / poetry / pytest / ruff, etc.)
  • Otherwise: minimal baseline (no hallucinated commands)
No command execution. No network. No AI inference.

Pack bundle (optional)

AI-readable docs bundle generated with the same safety model.

  • Crawl / index layer: llms.txt (or LLMS.md)
  • Execution layer: agents.entrypoints.json, docs/ai/how-to-run.md, docs/ai/how-to-test.md
  • Boundary layer: docs/ai/architecture.md, docs/ai/data-contracts.md
  • Handoff layer: docs/ai/id-context.json for ID-compatible workflows
  • Policy layer: SECURITY_AI.md, CONTRIBUTING_AI.md, README_SNIPPETS.md
Commands: agentsgen pack --autodetect and agentsgen pack --autodetect --check.
ID reads the human. agentsgen reads the repo. id-context.json connects them.
If markers are missing, pack writes sibling *.generated.* files instead of overwriting originals.

PR Guard + Pack drift check Recommended

Fail PRs when agent docs drift. Optionally enforce pack bundle drift in the same job.

Why it matters

  • Fail PRs when generated docs drift.
  • pack_check: "true" adds agentsgen pack --check.
  • Marker-only ownership stays intact for handcrafted docs.
Works on forks (best-effort comments).
Workflow 
name: agentsgen guard + pack check

on:
  pull_request:
  push:
    branches: [ main ]

permissions:
  contents: read
  pull-requests: write

jobs:
  agentsgen:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: markoblogo/AGENTS.md_generator/.github/actions/agentsgen-guard@main
        with:
          comment: "true"
          files: |
            AGENTS.md
            RUNBOOK.md
          pack_check: "true"
          pack_format: "json"
If you do not want PR comments, set comment: "false".

FAQ

Short answers.

What is AGENTS.md?
A repo-level contract for coding agents: setup, commands, guardrails, and “done” criteria. Think “README for agents”.
Will it overwrite my existing docs?
By default, no. Without markers it writes a sibling *.generated.md file and leaves your originals untouched.
What is the pack bundle?
A small bundle of AI-readable docs (run/test/architecture/contracts plus policies) that can be enforced via pack --check.
Does it work for monorepos?
It can detect mixed; if unsure, it stays conservative and may leave commands empty intentionally.

Known limitations

Conservative by design.

  • Monorepos may be detected as mixed; commands can be empty by design.
  • Unusual workflows should be set explicitly in .agentsgen.json.
  • This generates process guidance, not a full developer handbook.
