AGENTS.md Generator for AI coding agents

Generate and maintain AGENTS.md / RUNBOOK.md safely: marker-based updates, diff-first workflow, and conservative auto-detect of repo commands. Built for Claude Code, Codex, Cursor, and similar agents that need repo context fast.

Quickstart

Three commands. Diff-first workflow. No repo damage.

pipx install git+https://github.com/markoblogo/AGENTS.md_generator
cd your-repo
agentsgen init --autodetect
agentsgen update --dry-run --print-diff
agentsgen update
Run agentsgen check in CI to validate markers/config.

Use cases

  • Add repo context for Claude Code / Codex without rewriting docs by hand.
  • Keep agent docs consistent across many repos.
  • Prevent destructive edits: marker-only updates.
  • Standardize test/lint/build commands without “guessing”.

Problem / solution

AI agents are fast. Repo context is not.

Problem

Most repos either have no agent docs, or they’re outdated, inconsistent, and easy to break.

  • How to run tests / lint / build
  • What “done” means
  • What not to touch

Solution

agentsgen creates and updates agent docs using a safety-first model: updates only inside markers, otherwise generates a sibling file.

  • Markers + patch-by-range updates
  • Diff-first workflow (--dry-run, --print-diff)
  • Auto-detect (no AI)

Safety model

Safe to re-run: only generated sections are owned. Handcrafted docs stay yours.

1
File missing
Create AGENTS.md / RUNBOOK.md with section markers.
2
Markers exist
Patch-by-range: update only inside marker pairs.
3
No markers
Never overwrite. Write *.generated.md sibling files instead.

Auto-detect (no AI)

Priority rules. Conservative defaults.

  • Makefile (dominant)
  • package.json scripts
  • Python heuristics (uv / poetry / pytest / ruff, etc.)
  • Otherwise: minimal baseline (no hallucinated commands)
What it does not do: no command execution, no network, no AI inference.

FAQ

Short answers. No magic.

What is AGENTS.md?
A repo-level contract for coding agents: setup, commands, guardrails, and “done” criteria. Think “README for agents”.
Will it overwrite my existing docs?
By default: no. If markers are missing, the original file is left untouched and a sibling *.generated.md file is written instead.
How does autodetect work?
It only checks for cheap signals (files like Makefile, package.json, pyproject.toml, lockfiles). No command execution. No network.
Does it work for monorepos?
It can detect mixed and intentionally leave commands empty rather than hallucinate. In that case, provide explicit config in .agentsgen.json.
Why not just write it manually?
Manual docs drift. The marker model makes updates safe to re-run and keeps your custom content intact.

Known limitations

Conservative by design.

  • Monorepos may be detected as mixed, and commands can be left empty intentionally.
  • Unusual workflows should be defined explicitly in .agentsgen.json.
  • This generates process guidance, not a full developer handbook.
